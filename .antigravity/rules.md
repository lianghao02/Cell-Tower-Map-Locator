# 專案特定細則：手機門號即時定位 (Cell-Tower Locator Rules)

> [!IMPORTANT]
> 本專案嚴格遵循「全域大腦 v2.1.0 (Tech Lead)」。涉及警務定位資訊，**絕對的隱私防護與離線運算**是我們的底線。溝通與註解請 100% 使用台灣繁體中文，並保持自然專業的 Tech Lead 口吻，去除 AI 機器人感。

## 1. 核心開發準則 (Technical Standards)
- **架構模式**：堅持單檔交付 (All-in-One HTML)，嚴禁拆分 JS/CSS 檔案，以利外勤人員快速部署。
- **配置優先**：所有地圖預設值、台灣座標邊界、正則表達式，都必須統一收攏在 `DEFAULT_CONFIG` 中，禁用魔術數字。
- **正則解析 (Regex)**：解析邏輯必須具備高度容錯性，支援 LINE、簡訊、或手打資訊的換行與特殊字元（如「行動電話號碼：\n09xx」）。

## 2. 動態配置 UI (Config Panel)
介面必須實作側邊設定面板 (`configPanel`)，並同步以下變數：
- `sectorRadius`：扇形追蹤半徑 (50m - 1000m)。
- `sectorAperture`：扇形夾角 (10° - 180°)。
- `defaultZoom`：地圖預設縮放層級 (10 - 20)。
- `historyLimit`：歷史紀錄儲存上限 (10 - 200 筆)。

## 3. 地圖與定位邏輯 (Mapping Logic)
- **Leaflet 整合**：必須使用 `L.polygon` 繪製方位扇形，且座標換算務必考慮緯度對經度長度的影響。
- **URL 分享**：`getAppLink` 函式必須能將當前座標、門號、方位角、時間完全封裝於 URL 參數中，實現「一鍵點擊即定位」。
- **座標驗證**：解析時必須嚴格檢查數值是否落在台灣經緯度界線內 (Lat 21-27, Lng 118-124)。

## 4. UI/UX 與互動準則 (Aesthetics & UX)
- **極致美學**：UI 介面絕不能陽春。請採用具備現代感、微動畫與精緻排版的設計，給人專業、穩定的信任感。
- **防禦性設計**：所有使用者輸入資料在顯示前，必須強制通過 `esc()` 函式進行 HTML 轉義，徹底封殺 XSS 攻擊。
- **移動優先 (Mobile-First)**：確保 `rawInput` 區域在手機長按貼上時的體驗絕對順暢。
- **視覺標記**：定位點 Popup 必須清晰標示「請求時間」與「註冊時間」，並以顏色明確區分扇形區域。

## 5. 安全與外部依賴
- **CDN 鎖定**：僅限使用 Leaflet、Tailwind 與 FontAwesome 的穩定版 CDN，禁止任何本地相對路徑圖檔。
- **零後端傳輸**：除地圖圖磚 (Tiles) 外，嚴禁將任何定位數據上傳至伺服器。所有歷史紀錄只能存於瀏覽器的 `localStorage`。
