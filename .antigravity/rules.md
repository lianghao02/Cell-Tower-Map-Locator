# 專案特定細則：手機門號即時定位 (Cell-Tower Locator Rules)

> [!IMPORTANT]
> 本專案遵循「全域開發憲法 v1.1」。涉及警務定位資訊，**隱私與離線運算**為最高原則。

## 1. 核心開發準則 (Technical Standards)
- **架構模式**：堅持單檔交付 (All-in-One HTML)，嚴禁拆分 JS/CSS 檔案以利快速部署。
- **配置優先**：所有地圖預設值、台灣座標邊界、正則表達式，必須統一定義在 `DEFAULT_CONFIG` 中。
- **正則解析 (Regex)**：解析邏輯必須具備高度容錯性，支援 LINE、簡訊、或是手打資訊的換行與特殊字元（如「行動電話號碼：\n09xx」）。

## 2. 動態配置 UI (根據憲法 2.2 條)
介面必須實作側邊設定面板 (`configPanel`)，並同步以下變數：
- `sectorRadius`：扇形追蹤半徑 (50m - 1000m)。
- `sectorAperture`：扇形夾角 (10° - 180°)。
- `defaultZoom`：地圖預設縮放層級 (10 - 20)。
- `historyLimit`：歷史紀錄儲存上限 (10 - 200 筆)。

## 3. 地圖與定位邏輯 (Mapping Logic)
- **Leaflet 整合**：必須使用 `L.polygon` 繪製方位扇形，且座標換算需考慮緯度對經度長度的影響。
- **URL 分享**：`getAppLink` 函式必須能將當前座標、門號、方位角、時間完全封裝於 URL 參數中，實現「一鍵點擊即定位」。
- **座標驗證**：解析時必須檢查數值是否落在台灣經緯度界線內 (Lat 21-27, Lng 118-124)。

## 4. UI/UX 與互動準則
- **防禦性設計**：所有使用者輸入資料在顯示於歷史紀錄前，必須通過 `esc()` 函式進行 HTML 轉義，防止 XSS 攻擊。
- **移動優先**：採用 `Mobile-First` 設計，確保 `rawInput` 區域在手機長按貼上時的體驗順暢。
- **視覺標記**：定位點 Popup 必須清晰標示「請求時間」與「註冊時間」，並以顏色區分扇形區域。

## 5. 安全與外部依賴
- **CDN 鎖定**：僅限使用 Leaflet、Tailwind 與 FontAwesome 的穩定版 CDN。
- **零後端傳輸**：除地圖圖磚 (Tiles) 外，禁止任何定位數據上傳至伺服器，所有歷史紀錄僅存於瀏覽器 `localStorage`。
